### e.g. 1

```GLSL
uniform vec2 u_resolution;

void main(){
    vec2 st = gl_FragCoord.xy/u_resolution.xy;
    vec3 color = vec3(0.0);

    // Each result will return 1.0 (white) or 0.0 (black).
    float left = step(0.1,st.x);   // Similar to ( X greater than 0.1 )
    float bottom = step(0.1,st.y); // Similar to ( Y greater than 0.1 )

    // The multiplication of left*bottom will be similar to the logical AND.
    color = vec3( left * bottom );

    gl_FragColor = vec4(color,1.0);
}
```

The `step()` function will turn every pixel below 0.1 to black (`vec3(0.0)`) and the rest to white (`vec3(1.0)`) . The multiplication between `left` and `bottom` works as a logical `AND` operation, where both must be 1.0 to return 1.0 . This draws two black lines, one on the bottom and the other on the left side of the canvas.
In the previous code we repeat the structure for each axis (left and bottom). We can save some lines of code by passing two values directly to `step()` instead of one. That looks like this:

```glsl
vec2 borders = step(vec2(0.1),st);
float pct = borders.x * borders.y;
```
前一段是分开两次调用 `step`，分量是 float；这次是一次性给 `vec2` 调用 `step`

### e.g. 2

```GLSL
#ifdef GL_ES
precision mediump float;
#endif

uniform vec2 u_resolution;
uniform vec2 u_mouse;
uniform float u_time;

void main(){
    vec2 st = gl_FragCoord.xy/u_resolution.xy;
    vec3 color = vec3(0.0);

    // bottom-left
    vec2 bl = step(vec2(0.1),st);
    //vec2 bl = step(vec2(0.1,0.1),st.xy);
    
    //或者等价于
    //float bl_x = step(0.1, st.x); 
    //float bl_y = step(0.1, st.y); 
    //vec2 bl = vec2(bl_x, bl_y);
    float pct = bl.x * bl.y;

    // top-right
    vec2 tr = step(vec2(0.300,0.340),1.0-st);
    pct *= tr.x * tr.y;
    //pct = pct * (tr.x * tr.y);

    color = vec3(pct);

    gl_FragColor = vec4(color,1.0);
}
```

>[!note] 数学直觉
>- `float pct = bl.x*bl.y` 在视觉上是一个交集运算，等价于逻辑AND，由于初始化定义了画布颜色为vec3(0.0)即黑色，白色区域是主动构建的
>- 黑色部分看起来像st.xy<0.1的并集是因为符合德摩根定律：NOT(A∩B) = NOT(A) ∪ NOT(B)
>- 同理，如果取消line21-22的注释，构建的tr区域则是两个白色部分的交集，`pct *= tr.x * tr.y`这里的pct*里的乘法也是交集

lines 18 and 22 equivalent to writing:

```glsl
vec2 bl = step(vec2(0.1),st);       // bottom-left
vec2 tr = step(vec2(0.1),1.0-st);   // top-right
color = vec3(bl.x * bl.y * tr.x * tr.y);
```
依旧交集
