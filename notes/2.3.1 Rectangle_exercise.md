## Q1

>[!Question] Problem1
>Do another implementation that uses [`floor()`](https://thebookofshaders.com/glossary/?search=floor).

```GLSL
void main(){
    vec2 st = gl_FragCoord.xy/u_resolution.xy;

    vec2 grid = floor(st * 10.0);   // 把画面切成10x10的格子

    // 用mod判断格子编号的奇偶，奇偶交替染色，形成棋盘格
    float pattern = mod(grid.x + grid.y, 2.0);

    vec3 color = vec3(pattern);
    gl_FragColor = vec4(color, 1.0);
}
```
- `floor(st * 10.0)`：把 0~1 的连续坐标，切成 10×10 = 100 个格子，每个格子有自己的整数编号（比如左下角是(0,0)，往右一格是(1,0)……）
- `grid.x + grid.y`：把横纵格子编号加起来
- `mod(..., 2.0)`：取余2，结果只会是0或1，且**相邻格子的奇偶性必然相反**

## Q2
>[!question] Problem2
>Make another function that just draws the outline of a rectangle
### Solution1：outerRect - innerRect

```GLSL
float rect(vec2 st, vec2 bottomLeft, vec2 topRight) {
    vec2 bl = step(bottomLeft, st);
    vec2 tr = step(topRight, 1.0 - st);
    return bl.x * bl.y * tr.x * tr.y;
}

void main() {
    vec2 st = gl_FragCoord.xy / u_resolution.xy;

    float outerRect = rect(st, vec2(0.1), vec2(0.1));   // 大矩形（留白小）
    float innerRect = rect(st, vec2(0.12), vec2(0.12)); // 小矩形（留白大，所以矩形本身更小）

    float outline = outerRect - innerRect;   // 大减小，剩下一个细边框

    vec3 color = vec3(outline);
    gl_FragColor = vec4(color, 1.0);
}
```
### Solution2：abs

```glsl
float rectOutline(vec2 st, vec2 size, float thickness) {
    vec2 bl = step(size, st);              // 距离左/下边界的判断
    vec2 tr = step(size, 1.0 - st);        // 距离右/上边界的判断
    float outer = bl.x * bl.y * tr.x * tr.y;  // 大矩形（外边界）

    vec2 bl2 = step(size + thickness, st);
    vec2 tr2 = step(size + thickness, 1.0 - st);
    float inner = bl2.x * bl2.y * tr2.x * tr2.y;  // 小矩形（内边界）

    return outer - inner;
}
```

### Solution3:SDF

```GLSL
float rectOutline(vec2 st, vec2 size, float thickness) {
    vec2 d = abs(st - vec2(0.5)) - size;   // 计算到矩形边界的"有符号距离"
    float outer = max(d.x, d.y);            // 取较大的那个轴向距离

    return (outer < 0.0 && outer > -thickness) ? 1.0 : 0.0;
}
```

 `vec2 d = abs(st - vec2(0.5)) - size;`

分两步看：

**第一步：`st - vec2(0.5)`** —— 把坐标原点从左下角，移到画面正中心。

```
st=(0.5,0.5) (中心)   → 变成 (0, 0)
st=(0.8,0.5) (右边)   → 变成 (0.3, 0)
st=(0.2,0.5) (左边)   → 变成 (-0.3, 0)
```

**第二步：`abs(...)`** —— 取绝对值，把左右两侧、上下两侧都折叠到同一边，**这样只需要写一套判断逻辑，就能同时覆盖四个方向**

```
变成(0.3, 0) 或 (-0.3, 0)，取绝对值后都变成 (0.3, 0)
```

**第三步：减去 `size`** —— `size` 是矩形半边长（比如 `vec2(0.3)` 表示矩形从中心往外延伸0.3的范围）。这一步算出的是：**当前点，相对"矩形边界"，还差多远**。

```
如果点在矩形边界正上 → d.x或d.y恰好等于0
如果点在矩形内部     → d.x和d.y都是负数（说明"还没到边界，是负的余量"）
如果点在矩形外部     → 至少有一个是正数（说明"超出边界了"）
```

 `float outer = max(d.x, d.y);`

矩形有四条边，每个点其实"分别"对横向边界、纵向边界都有一个距离值。**取两者较大的那个**，意思是"看这个点离矩形最近的那条边到底有多远"——这是判断"是否在矩形内"的标准技巧：**只要横纵两个方向都没超界（都是负数），这个点才真正在矩形内部**；只要任意一个方向超界了（变成正数），取max之后结果也会变成正数，说明这个点已经跑出矩形了。

```
举例：某点 d = (-0.05, 0.1)
        ↑横向还在矩形内(负)  ↑纵向已经超出矩形了(正)
outer = max(-0.05, 0.1) = 0.1   → 正数，说明这个点确实在矩形外部（被纵向超界拖累了）
```

 `return (outer < 0.0 && outer > -thickness) ? 1.0 : 0.0;`

这一行才是真正画出"轮廓"（而不是"实心矩形"）的关键：

```
outer < 0.0          → 在矩形内部（基本判断）
outer > -thickness   → 但没有深入太多，还在边界往里数 thickness 这么的一层范围内
```

```
       超出矩形(outer>0)
┌─────────────────────┐
│ ░░░ 边框区域 ░░░░░░░ │ ← -thickness < outer < 0，这一圈是1（被选中）
│ ░░┌───────────────┐░ │
│ ░░│               │░ │
│ ░░│   矩形内部更深  │░ │ ← outer < -thickness，太深了，不算边框，返回0
│ ░░│   (outer更负)  │░ │
│ ░░└───────────────┘░ │
└─────────────────────┘
```

只有那一圈"刚好在边界附近、深度不超过thickness"的区域，才会判定为1（白色），其余无论是矩形内部更深的地方，还是矩形外面，都是0（黑色）。
#### 和前两种思路的关系

本质上，**`outer` 这个值，就是一种简化版的SDF（带符号距离）**——负数代表在形状内部，正数代表在外部，数值大小代表"离边界有多远"。这一种写法之所以更紧凑，是因为它**直接算出了"离边界的距离"，不需要像思路一那样真的构造两个独立的矩形再做减法**，而是一步到位地问"这个点离最近的边界够不够近"。
