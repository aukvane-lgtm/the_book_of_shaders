>[!note] 核心思路：SDF 思维（距离场判断）
  整章贯穿的思路基本是同一个套路：
> 
> ```
> 对每一个像素，计算它和某个"目标形状"的距离/位置关系
>         ↓
> 用这个距离/关系，决定这个像素该不该被点亮（该不该上色）
> ```
> 
> 这是[[2.1 shaping functions|2.1节]]的 `plot` 函数思路的扩展版——之前是"判断像素离一条线有多近"，这一章会扩展到"判断像素是否落在圆形/矩形/多边形内部"。

## 会用到的主要工具

### 1. `step()` ：硬边

```glsl
float pct = step(0.5, st.x);
```

- 两个参数：**阈值（边界）** 和 **要检测的值**，小于阈值返回 `0.0`，大于等于阈值返回 `1.0`，硬边缘
### 2. `smoothstep()` ：羽化边缘

```glsl
float pct = smoothstep(edge0, edge1 , x);
```
- 把 `step()` 的硬边"软化"，虚化边缘。

### 3. `length()` ：画圆形

```glsl
float d = length(st - vec2(0.5));   // 当前点到圆心的距离
```

圆形的本质定义就是"到圆心距离恒定的所有点"，所以 `length()` 几乎是画圆形必用的函数，在[[2.2 Color_HSB的极坐标转换|2.2章]]色轮那段代码里已经接触过它的用法（`radius = length(toCenter)`）。

### 4. `atan()` ：配合极坐标，画多边形、星形等放射状图案

依旧是2.2章用过的（色轮那段代码用 `atan` 算角度），本章会延伸出"用角度配合距离，画出三角形、五角星、花瓣"等更复杂图形的技巧。

### 5. 距离场的组合运算（min / max）—— 多个形状的合并/相减

```glsl
float d = min(circle1, circle2);   // 两个圆形"并集"
float d = max(circle1, circle2);   // 两个圆形"交集"
```

这是这一章会教的新内容——**怎么把简单形状组合成复杂形状**，用 `min`/`max` 模拟"图形的加减法"（类似图形之间的布尔运算）

![[Pasted image 20260624141525.png|不如这样想：illustrator的路径查找器|200]]

| Illustrator              | 视觉效果                | Shader                       |
| ------------------------ | ------------------- | ---------------------------- |
| **交集（Intersect）**        | 只留重叠部分              | `a * b`（相乘）                  |
| **联集（Unite）**            | 合并两个形状              | `max(a, b)` 或 `a + b - a*b`  |
| **差集（Minus Front）**      | A减去B                | `a * (1.0-b)`                |
| **差集（仅保留不重叠部分，Exclude）** | 排除掉重叠区域，留下"各自独有"的部分 | `abs(a - b)` 或 `a+b-2.0*a*b` |

## Summary

1. 怎么用数学定义"什么样的点算在形状里面"
2. 怎么把这个"是否在里面"的判断，转换成 0~1 的颜色值（复用plot/mix 套路）
3. 怎么通过组合、变形这些基础形状，搭出更复杂的图案
