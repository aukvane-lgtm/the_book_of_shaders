```glsl
#ifdef GL_ES
precision mediump float;
#endif

#define TWO_PI 6.28318530718

uniform vec2 u_resolution;
uniform float u_time;

//  Function from Iñigo Quiles
//  https://www.shadertoy.com/view/MsS3Wc
vec3 hsb2rgb( in vec3 c ){
    vec3 rgb = clamp(abs(mod(c.x*6.0+vec3(0.0,4.0,2.0),
                             6.0)-3.0)-1.0,
                     0.0,
                     1.0 );
    rgb = rgb*rgb*(3.0-2.0*rgb);
    return c.z * mix( vec3(1.0), rgb, c.y);
}

void main(){
    vec2 st = gl_FragCoord.xy/u_resolution;
    vec3 color = vec3(0.0);

    // Use polar coordinates instead of cartesian
    vec2 toCenter = vec2(0.5)-st;
    float angle = atan(toCenter.y,toCenter.x);
    float radius = length(toCenter)*2.0;

    // Map the angle (-PI to PI) to the Hue (from 0 to 1)
    // and the Saturation to the radius
    color = hsb2rgb(vec3((angle/TWO_PI)+u_time*0.1,radius,1.0));

    gl_FragColor = vec4(color,1.0);
}
```

这段代码在前一个的基础上，加入了**极坐标变换**，把"色相条渐变图"变成了一个**彩虹色轮（color wheel）**。我们重点讲新增的部分。

## 已经认识的部分

`hsb2rgb` 函数和上一个例子完全一样，不再重复讲。`#define TWO_PI 6.28318530718` 定义了 2π 这个常数，用来做弧度换算。

---

## 核心新内容：从笛卡尔坐标转极坐标

```glsl
vec2 st = gl_FragCoord.xy/u_resolution;
vec2 toCenter = vec2(0.5) - st;
float angle = atan(toCenter.y, toCenter.x);
float radius = length(toCenter) * 2.0;
```

这四行做的事，是把"每个像素在画面上的位置"，从**直角坐标（x, y）**，转换成**极坐标（角度, 半径）**——这是制作圆形/放射状图案的标准套路，之前两个例子都是用 `st.x`/`st.y` 直接映射颜色，这次换了一种"以画面中心为原点"的描述方式。

### 第一步：`toCenter = vec2(0.5) - st`

`st` 的范围是 `[0,1]`，画面正中心是 `(0.5, 0.5)`。

```glsl
toCenter = vec2(0.5) - st
```

意思是：**算出"画面中心"到"当前像素"的方向向量**。

举几个例子：

|像素位置 st|toCenter|含义|
|---|---|---|
|(0.5, 0.5)|(0, 0)|正中心，向量长度=0|
|(1.0, 0.5)|(-0.5, 0)|画面最右边，指向左边（因为中心-右边=向左的向量）|
|(0.5, 1.0)|(0, -0.5)|画面最上边，指向下方|
|(0.0, 0.0)|(0.5, 0.5)|左下角，指向右上方|
### 第二步：`angle = atan(toCenter.y, toCenter.x)`

`atan(y, x)` 是**双参数反正切函数**（在数学里常写作[[atan2]]），它的作用是：**把一个二维向量 `(x,y)` 转换成这个向量指向的角度**。

```
atan(y, x) 返回的角度范围是 [-π, π]
        π/2 (正上方)
         |
	π────0 ──── -π(正右方)
         |
       -π/2 (正下方)
```

对照刚才的例子：

|toCenter|angle (弧度)|角度方向|
|---|---|---|
|(-0.5, 0)|π（或-π，边界）|指向左边|
|(0, -0.5)|-π/2|指向下方|
|(0.5, 0)|0|指向右边|
|(0, 0.5)|π/2|指向上方|

也就是说，**`angle` 描述了"当前像素相对画面中心，在哪个方向上"**。如果你站在画面中心，朝当前像素看过去，`angle` 就是你转过的角度。

### 第三步：`radius = length(toCenter) * 2.0`

`length(v)` 计算向量的长度（也就是勾股定理 `sqrt(x²+y²)`），这里算的是**当前像素到画面中心的距离**。

```
中心点 (0,0)        → radius = 0
画面边缘最远点(0.5,0) → length=0.5，乘2.0后 → radius = 1.0
```

乘以 `2.0` 是为了把范围从 `[0, 0.5]` 拉伸到 `[0, 1.0]`，这样后面用 `radius` 当饱和度（S，范围需要是0~1）时刚好对上。

---

## 把角度和半径映射到颜色

```glsl
color = hsb2rgb(vec3((angle/TWO_PI)+0.5, radius, 1.0));
```

### `(angle/TWO_PI) + 0.5` → 映射成色相 H

`angle` 范围是 `[-π, π]`，除以 `TWO_PI`（即2π）后变成 `[-0.5, 0.5]`，再加 `0.5` 平移成 `[0, 1.0]`——刚好符合色相 H 需要的 0~1 范围。

效果：**围绕画面中心转一圈（360°），颜色会经历完整的一轮色相变化**（红→黄→绿→青→蓝→紫→红）。

### `radius` → 直接作为饱和度 S

距离中心越远，饱和度越高（`radius` 从0到1）。

效果：**画面中心是白色（饱和度=0，无论色相是什么，都会被白化），越往外围颜色越纯越鲜艳**。

### `1.0` → 明度 B 固定拉满

不随位置变化，整张图都保持最大亮度。
